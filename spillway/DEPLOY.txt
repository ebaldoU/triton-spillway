GUÍA DE DESPLIEGUE — SPILLWAY APP
==================================
Sustituir <servidor>, <usuario> y <ruta-datos> por los valores reales.


1. ESTRUCTURA DE DIRECTORIOS EN EL SERVIDOR
--------------------------------------------
/home/<usuario>/
  spillway/              <- scripts de la app
    app.py
    consultas_analiticas.py
    config.py
    geotiff_to_tiledb_sparse.py
    verify_tiledb.py
    export_to_geotiff.py
  <ruta-datos>/
    triton_results/      <- arrays TileDB (~17-21 GB por dataset)
      output_10_..._<fechas>/   (datos1)
      output_10_..._<fechas>/   (datos2)
      ...
  venv_triton/           <- entorno Python


2. PRIMERA INSTALACIÓN (solo una vez)
--------------------------------------

En el servidor:

  # Instalar dependencias del sistema
  sudo apt update && sudo apt upgrade -y
  sudo apt install -y python3-pip python3-venv python3-full libgdal-dev

  # Crear entorno virtual
  python3 -m venv ~/venv_triton
  source ~/venv_triton/bin/activate

  # Instalar paquetes Python
  pip install tiledb streamlit plotly folium streamlit-folium pyproj numpy rasterio pandas Pillow

  # Crear directorios
  mkdir -p ~/spillway <ruta-datos>/triton_results


Desde la máquina local, subir scripts:

  rsync -avz \
    /ruta/local/spillway/app.py \
    /ruta/local/spillway/consultas_analiticas.py \
    /ruta/local/spillway/config.py \
    /ruta/local/spillway/geotiff_to_tiledb_sparse.py \
    /ruta/local/spillway/verify_tiledb.py \
    /ruta/local/spillway/export_to_geotiff.py \
    <usuario>@<servidor>:/home/<usuario>/spillway/

Desde la máquina local, subir datos TileDB (tarda varias horas según el volumen):

  rsync -avz --progress --delete \
    /ruta/local/triton_results/ \
    <usuario>@<servidor>:<ruta-datos>/triton_results/


3. SERVICIO SYSTEMD (arranque automático)
------------------------------------------

Archivo a crear en el servidor:
  sudo nano /etc/systemd/system/spillway.service

Contenido del archivo:

  [Unit]
  Description=Spillway - Flood Simulation Dashboard
  After=network.target

  [Service]
  User=<usuario>
  WorkingDirectory=/home/<usuario>/spillway
  Environment="TRITON_BASE_URI=<ruta-datos>/triton_results"
  Environment="SPILLWAY_USER=triton"
  Environment="SPILLWAY_PASS_HASH=<hash-sha256-de-la-contrasena>"
  ExecStart=/home/<usuario>/venv_triton/bin/streamlit run app.py --server.port 8501 --server.headless true --server.address 127.0.0.1
  Restart=on-failure
  RestartSec=5

  [Install]
  WantedBy=multi-user.target

NOTA: la app trae credenciales por defecto triton / demo. Las dos variables
Environment de arriba son opcionales y solo sirven para usar credenciales
propias en produccion (tienen prioridad sobre los valores por defecto).
Generar el hash de una contrasena propia con:

  python -c "import hashlib; print(hashlib.sha256('MI_CONTRASENA'.encode()).hexdigest())"

y pegar el resultado en la linea Environment de SPILLWAY_PASS_HASH.

Activar el servicio (solo la primera vez):

  sudo systemctl daemon-reload
  sudo systemctl enable spillway
  sudo systemctl start spillway
  sudo systemctl status spillway   <- debe mostrar "active (running)"


4. ACCESO A LA APP
-------------------

La app no está expuesta a internet directamente.
Para acceder se usa un túnel SSH desde la máquina local:

  ssh -L 8501:localhost:8501 <usuario>@<servidor>

Mientras ese terminal esté abierto, abrir en el navegador:

  http://localhost:8501

Al cerrar el terminal el túnel se corta, pero la app sigue corriendo en el servidor.


5. COMANDOS ÚTILES EN EL SERVIDOR
-----------------------------------

Ver estado de la app:
  sudo systemctl status spillway

Reiniciar la app:
  sudo systemctl restart spillway

Ver logs en tiempo real:
  sudo journalctl -u spillway -f

Parar la app:
  sudo systemctl stop spillway

Actualizar scripts (desde la máquina local):
  rsync -avz \
    /ruta/local/spillway/app.py \
    /ruta/local/spillway/consultas_analiticas.py \
    /ruta/local/spillway/config.py \
    <usuario>@<servidor>:/home/<usuario>/spillway/
  ssh <usuario>@<servidor> "sudo systemctl restart spillway"


6. AÑADIR UN DATASET NUEVO
----------------------------

Desde la máquina local:
  rsync -avz --progress --delete \
    /ruta/local/triton_results/<nombre_nuevo>/ \
    <usuario>@<servidor>:<ruta-datos>/triton_results/<nombre_nuevo>/

Si el nuevo dataset tiene alias corto (p.ej. datos57), añadirlo en config.py:
  "datos57": "<nombre_largo>",

Luego actualizar config.py en el servidor y reiniciar:
  rsync -avz /ruta/local/spillway/config.py <usuario>@<servidor>:/home/<usuario>/spillway/
  ssh <usuario>@<servidor> "sudo systemctl restart spillway"
